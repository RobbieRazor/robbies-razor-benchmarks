# Recursive Blast Radius Limit

The Recursive Blast Radius Limit describes the maximum architectural depth
that recursive modification may penetrate before systemic stability degrades.

Recursive systems typically organize into layers:

Application
Service Dependencies
Orchestration
Infrastructure
Compute Substrate

Recursion confined to upper layers produces localized instability.

Recursion crossing infrastructure layers increases systemic instability.

Define the Instability Gradient:

G = ΔInstability / ΔLayerDepth

As recursion approaches substrate layers, instability grows nonlinearly.

The boundary where recursion begins modifying the mechanisms that enable
recursion itself is defined as the Recursive Event Horizon.

Beyond this point recovery mechanisms become unreliable.

Implication:

Stable recursive systems regulate recursion depth rather than
merely increasing computational scale.
