# Recursion Under Constraint

Recursive systems operate within two structural ceilings defined in **MRD §11**.

## Energetic Recursion Ceiling

Energy availability limits recursive transitions.

R ≤ E / JCT

Where:

- **E** = available energy per unit time  
- **JCT** = Joules per Coherent Transition  
- **R** = recursive transition rate  

Compression-efficient architectures reduce **JCT**, allowing more recursion within the same energy budget.

---

## Governance Recursion Ceiling

Stabilization capacity also constrains recursion velocity.

R · C ≤ S

Where:

- **S** = stabilization bandwidth  
- **C** = correction demand per transition  

Instability occurs when recursive transitions generate correction demand faster than stabilization capacity can absorb.

---

# Safe Recursion Envelope

Stable systems must satisfy both ceilings simultaneously.

R ≤ min(E/JCT , S/C)

This region defines the **Safe Recursion Envelope**.

Systems operating outside this envelope enter structural instability regimes.

---

# Threshold Compression Gain

Recursive intelligence systems often appear to improve slowly and then suddenly accelerate.

This occurs when compression improvements allow a system to cross a recursion ceiling boundary.

Small reductions in:

- Joules per Coherent Transition (JCT)
- correction demand per transition (C)
- recomputation burden

can unlock disproportionately large increases in recursive throughput.

This phenomenon is called **Threshold Compression Gain**.

It explains why capability growth may appear discontinuous even when underlying improvements are gradual.
