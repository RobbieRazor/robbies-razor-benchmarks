# Core Idea

Robbie’s Razor proposes that stable intelligence systems follow the cycle:

compression → expression → memory → recursion

Where:

- **Compression** extracts stable structure from information  
- **Expression** reconstructs useful representations  
- **Memory** preserves compressed structure  
- **Recursion** repeatedly re-enters preserved structure to generate predictions and actions

Systems that violate this architecture typically compensate by expanding external boundaries such as compute, energy, or infrastructure.

# Constraint Law

Recursive intelligence is bounded by two independent ceilings:

Energetic Recursion Ceiling

R ≤ E / JCT

Governance Recursion Ceiling

R · C ≤ S

Where:

- **E** = available energy
- **JCT** = Joules per coherent recursive transition
- **S** = stabilization bandwidth
- **C** = correction demand per transition
- **R** = recursion rate

Stable systems must operate inside the **Safe Recursion Envelope**:

R ≤ min(E/JCT , S/C)

# Threshold Compression Gain

Recursive systems often appear to improve slowly and then suddenly accelerate.

This occurs when compression improvements allow a system to cross a recursion ceiling boundary.

Small reductions in:

- recomputation burden
- Joules per Coherent Transition
- correction demand

can unlock large increases in recursive throughput.

This phenomenon is called **Threshold Compression Gain**.
