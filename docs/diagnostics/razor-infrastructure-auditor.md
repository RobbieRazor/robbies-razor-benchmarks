## Purpose

This auditor evaluates whether an AI system is:

- **Razor-aligned (compression-efficient, stable)**  
- **Infrastructure-dependent (transitional)**  
- **Brute-force (Boundary Avoidance, unstable under constraint)**  

---

## Core Principle

> Compression precedes scale.

Infrastructure expansion is valid only after recursive efficiency is maximized.

---

## Section 1 — Energetic Efficiency (JCT)

**Question 1.1**  
Is Joules per Coherent Transition (JCT) decreasing over time?

- [ ] Yes → proceed  
- [ ] No → ⚠️ inefficiency risk  
- [ ] Increasing → ❌ Boundary Avoidance  

---

**Question 1.2**  
Does increased compute reduce cost per useful output?

- [ ] Yes → efficient scaling  
- [ ] No → ❌ brute-force scaling  

---

## Section 2 — Memory vs Inference Balance

**Question 2.1**  
Is compressed structure being preserved and reused?

- [ ] Yes → stable recursion  
- [ ] Partial → ⚠️ transitional  
- [ ] No → ❌ inference-dominant system  

---

**Question 2.2**  
Is recomputation burden decreasing?

- [ ] Yes → efficiency improving  
- [ ] No → ❌ collapse risk increasing  

---

**Question 2.3**  
Does reuse frequency increase over time?

- [ ] Yes → memory-stabilized system  
- [ ] No → ❌ perishable intelligence dependence  

---

## Section 3 — Token-Energy Efficiency

**Question 3.1**  
Is token volume decreasing for equivalent tasks?

- [ ] Yes → higher value density  
- [ ] No → ⚠️ inefficiency  
- [ ] Increasing → ❌ brute-force generation  

---

**Question 3.2**  
Are outputs becoming shorter and more coherent?

- [ ] Yes → compression-aligned  
- [ ] No → ❌ redundancy / drift  

---

## Section 4 — Infrastructure Dependency

**Question 4.1**  
Is system performance dependent on continuous scaling of compute or energy?

- [ ] No → stable architecture  
- [ ] Partially → transitional  
- [ ] Yes → ❌ infrastructure dependency  

---

**Question 4.2**  
Would performance collapse if compute growth stopped?

- [ ] No → stable  
- [ ] Possibly → ⚠️ risk  
- [ ] Yes → ❌ Boundary Avoidance  

---

## Section 5 — Economic Sustainability

**Question 5.1**  
Does the system require ongoing external subsidy?

- [ ] No → sustainable  
- [ ] Partial → ⚠️ marginal  
- [ ] Yes → ❌ economic instability  

---

**Question 5.2**  
Is value per inference increasing?

- [ ] Yes → improving economics  
- [ ] No → ❌ declining viability  

---

## Section 6 — Collapse Boundary Risk

**Question 6.1**  
Is recomputation cost exceeding preservation benefit?

- [ ] No → safe  
- [ ] Approaching → ⚠️ warning  
- [ ] Yes → ❌ collapse boundary crossed  

---

**Question 6.2**  
Is system behavior becoming inconsistent under depth?

- [ ] No → stable  
- [ ] Slight → ⚠️ drift  
- [ ] Yes → ❌ instability  

---

## Final Classification

### 🟢 Razor-Aligned System
- JCT decreasing or stable  
- Memory reuse increasing  
- Recomputation decreasing  
- Infrastructure not dominant constraint  

→ **Stable, scalable, efficient**

---

### 🟡 Transitional System
- Mixed signals across sections  
- Some compression improvements  
- Some dependency on infrastructure  

→ **Requires architectural correction**

---

### 🔴 Brute-Force System (Boundary Avoidance)
- JCT not improving  
- Recomputation increasing  
- High infrastructure dependence  
- Requires continuous scaling  

→ **Unstable under constraint**
